01
The Challenge

What Marriott Told Us

Across our discovery conversations, five clear themes emerged. Each one points to the same gap: a critical data protection and access layer that needs to be in place before Loyalty Cloud goes live.

Select a theme to expand

Need for a "Warm Storage" Solution
Marriott needs an archiving layer that keeps Salesforce performant while still allowing on-demand access to full member history when service agents need it. Storing everything in the core org is not viable at Bonvoy scale.
Strong Push Toward Self-Service Data Access
Marriott's team is actively working to reduce reliance on data brokers and external system hand-offs. The vision is a single-pane experience — no jumping to Tableau, no submitting data requests — everything accessible directly from Salesforce.
Loyalty-Specific Requirements
Full member history visibility is non-negotiable — even for data stretching back years. Missing points disputes, tier status challenges, and partner activity history all require complete historical context to resolve correctly.
Enterprise Constraints & Nuances
Certain member tiers carry a 50-year data retention requirement — far beyond standard enterprise data management. The existing ecosystem (Snowflake, Adobe, and others) must be complemented, not replaced.
Must-Have: Disaster Recovery Before Go-Live
Backup & Recover is a hard requirement before Loyalty Cloud enters production. Marriott cannot risk a corruption event, operational mistake, or cyber incident against 271M member records without a validated recovery mechanism.
"Disaster recovery isn't optional — it has to be in place before we go live."

Data Lifecycle Strategy: HOT → WARM → COLD → BACKUP
ARCHITECTURE ▼

Salesforce Backup + Archive enables a complete data lifecycle strategy for Marriott's Loyalty Cloud — ensuring performance at scale while maintaining historical access and regulatory compliance.

HOT — Active Core Org
  • Current member profiles and active transactions
  • Live loyalty points, tier status, active cases
  • Optimized for transactional performance
WARM — Archive Layer
  • Historical data accessible on demand
  • Surfaced within Salesforce UI — no separate tool
  • Preserves org performance at 271M+ scale
COLD — Long-Term Retention
  • 50-year retention for qualifying member tiers
  • Meets regulatory and compliance requirements
  • Accessible via Archive when needed
BACKUP — Recovery Layer
  • Daily automated backups of all Loyalty data
  • Point-in-time recovery capability
  • DR3 exercises validate playbook before go-live
